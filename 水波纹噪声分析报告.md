## 1. 问题描述

在 ACE 检测的 **score map**（分数图）中，观察到了显著的同心圆水波纹噪声：

| 现象 | 描述 |
|------|------|
| **形状** | 从中心向外扩散的同心圆环，类似水面涟漪 |
| **多中心** | 不同波段观察到不同的波纹中心点 |
| **动态感** | 逐波段观察时，波纹中心点位置"漂移移动" |
| **影响** | 大量虚假检测像素，降低目标检测可信度 |

## 3. 噪声产生机理

### 3.1 核心机制图解

```
 ┌─────────────────────────────────────────────────────────────────────┐
 │                    水波纹噪声产生机制                                 │
 └─────────────────────────────────────────────────────────────────────┘

 1. 根源: 光学路径中的干涉条纹
    ─ 在 Sky/Reflectance 图像中存在 17.5px 固定周期
    ─ 可能来源: 镜头元件间的法布里-珀罗干涉 (etalon effect)
    ─ 或: 微透镜阵列 / 光学窗口的薄膜干涉

 2. 棱镜色散: dX/dλ = 0.09 px/nm
    ─ 93 个波段横跨 ΔX ≈ 42 px 的 sensor 区域
    ─ 同一场景点的不同波段数据来自 sensor 不同列

 3. 数据流中的传递:
    反射率 R(x,y,λ) = (Spec − Dark) / (Sky − Dark)
    ─ Sky 的周期性非均匀性被除法传递到反射率中
    ─ 各波段的周期性相位 ∝ (X₀ + 0.09·λ) / 17.5px

 4. ACE 检测中的放大:
    ACE(x) = (dᵀR⁻¹(x-μ))² / ((dᵀR⁻¹d)·(x-μ)ᵀR⁻¹(x-μ))
    ─ 协方差逆矩阵 R⁻¹ 放大光谱维的周期性
    ─ 93 个波段的不同相位 → 空间同心圆图案
    ─ 不同波段"中心"不同 → 用户观察到的"波纹中心漂移"
```

### 3.2 棱镜色散：中心漂移的证据

![中心漂移](output/ace_samples/noise_center_vs_wavelength.png)
*左：不同波段（band 0 紫色 → band 92 红色）的 FFT 峰值相位漂移  
右：不同 band 在同一位置的列剖面，显示相位随波段系统性变化  
→ 证明棱镜色散将空间周期映射为光谱维度变化*

**定量数据**：
- dX/dλ = **0.0908 px/nm**
- 全谱 93 波段总漂移 = **41.8 px** ≈ **2.5 × 17.5px**
- 这意味着不同波段之间最多有 **2.5 个周期的相位差**
- 正是这个相位差在 ACE 协方差逆运算中被转化为同心圆结构

### 3.3 为什么暗场没有这个噪声？

![暗场分析](output/ace_samples/noise_dark_analysis.png)
*暗场图像的全频段 FFT 分析：2048/n 谐波来自读出时钟（电子噪声），但**无 17.5px 峰**  
→ 证明 17.5px 周期是光学干涉，不是传感器电子噪声*

## 4. 排除其他噪声源

### 4.1 法布里-珀罗干涉 (Etalon) 测试

![Etalon 测试](output/ace_samples/noise_etalon_test.png)
*左图：各波段的 FFT 幅值变化；右图：周期随波长变化  
Etalon 干涉的周期应严格随波长线性变化 (Period × λ = constant)*

| 检验项目 | 结果 | 判定 |
|---------|------|------|
| 周期在空间固定 | 各波段 17.5px 周期稳定 | ✅ 空间固定，非光谱干涉 |
| Period × λ CV | **26.9%** > 10% | ❌ **排除**标准 Etalon |
| 暗场存在性 | 暗场无 17.5px | ❌ 排除传感器读出噪声 |
| 光学模拟 | 17.5px ≈ 60μm | ✅ 可能为镜头/窗口薄膜干涉 |

**最终判定**: 噪声源是光学路径中某元件的**固定干涉条纹**（如滤波片、窗口片的薄膜干涉），通过棱镜色散被映射到光谱维度。

### 4.2 其他排除项

| 噪声类型 | 排除理由 |
|---------|---------|
| 传感器读出噪声 | 暗场中无 17.5px 周期 |
| 微透镜阵列图案 | 17.5px≈60μm，远大于 typical MLA pitch (~5-10μm) |
| Moiré 采样伪影 | 周期与像素采样率无关，谐波完整 |
| 场景自身纹理 | 所有场景（Sky/Spec/Gray）均存在 |

## 5. 解决方案：三级级联空间频域陷波滤波

### 5.1 方案设计

基于噪声机理分析，设计 **三级级联滤波** 策略，在数据流各环节分别抑制噪声：

```
  ┌─────┐    ┌──────────────┐    ┌──────────────────┐    ┌──────────────┐
  │     │    │  ① Sky 预滤波   │    │  ② Reflectance    │    │  ③ Score Map  │
  │ Raw │───→│   (2D FFT陷波)  │───→│  逐波段列滤波     │───→│  后处理平滑   │
  │     │    │  去除源头干涉    │    │  (1D列FFT×93波段) │    │  (中值滤波)   │
  └─────┘    └──────────────┘    └──────────────────┘    └──────────────┘
                                                     
                    ↓                        ↓                      ↓
                 移除 Sky 中的              消除反射率中           平滑后输出中
                 光学干涉条纹               的残留周期性            的残余噪声
```

### 5.2 陷波滤波器设计

**高斯窄带陷波** — 避免矩形窗的振铃伪影：

$$H(k) = 1 - A \cdot \exp\left(-\frac{(k - k_0)^2}{2\sigma^2}\right)$$

| 参数 | 值 | 说明 |
|------|-----|------|
| 陷波频率 | 1/17.5, 1/8.8, 1/5.9 px⁻¹ | 对应基频和 2/3 次谐波 |
| 带宽 σ | **0.005 px⁻¹** (~3px 频域半宽) | 窄带，只影响噪声频率 |
| 衰减 A | **0.95** (保留5%能量) | 避免完全移除导致的振铃 |

### 5.3 滤波器频率响应

![频率响应](output/filter_response.png)

*陷波滤波器在 0.0571 px⁻¹ (17.5px)、0.1136 px⁻¹ (8.8px)、0.1695 px⁻¹ (5.9px) 处有窄带衰减*

### 6.1 FFT 频域对比 — 噪声被彻底抑制

> 反射率中的 17.5px 周期几乎被完全消除（幅度从 7 千万降至 10），证明二级级联滤波极其有效。

### 6.2 Score Map 对比 — 同心圆消失

![Score Map 对比](output/filter_comparison/score_map_comparison.png)

- **左/左上**: 滤波前 → 清晰可见同心圆水波纹

### 6.3 检测结果对比 — 虚假检测减少（匹配目标模板）

> ⚠️ **重要修复**: 最初的对比使用了未滤波的原始目标模板 (`HI/target{1-3}.npy`) 去检测滤波后的场景，导致目标与场景不匹配，检测数从 89,659 错误降至 7,402（人为的过度抑制）。修复后，目标模板从滤波后的反射率中重新提取（使用 `id_to_key.json` + `mask.npy` + `coords_dict.json` 标注文件），确保检测时目标和场景使用同源数据。

| 检测像素数 | 89,659 | **72,559** | ↓ **19.1%** ✅ |

## 9. 结论

### ✅ 核心成果

1. **噪声源头已定位**: 光学路径中的固定干涉条纹（17.5px 周期）通过棱镜色散被映射为光谱维周期性 → ACE 协方差逆放大为同心圆水波纹

2. **滤波器已实现**: 三级级联陷波滤波 (Sky预滤波 → Reflectance逐波段滤波 → Score Map后处理) 无需修改传感器硬件

3. **效果验证**:
   - ✔ 17.5px 频域幅度抑制 **95-100%**
   - ✔ Score Map 同心圆噪声 **完全消失**
   - ✔ 目标模板匹配修复: 滤波后的目标需从滤波后反射率中提取
   - ✔ 综合最佳方案: **SACE + Notch Filter(Full) + med_open后处理**
   - ✔ 虚假检测有效减少，目标区域基本保留

4. **适用性**: 全部 5 种检测算法（ACE/CEM/SAM/SACE/MTICEM）均受益

### 📈 新检测基准方案

| 配置 | 说明 | 使用方式 |
|------|------|---------|
| 检测算法 | **SACE**（自适应余弦+角度约束） | `--method SACE --filter full` |

### 🏆 各算法适用场景建议

| 场景 | 推荐算法 | 理由 |
|------|---------|------|
| **通用目标检测** | **SACE** + Notch + med_open | 综合最佳，噪声低+检测区域合理+最大分保留 |
| **弱小目标检测** | **ACE** + Notch + med_open | 灵敏度最高，检测区域略多于SACE |
| **背景纯净优先** | **CEM** + Notch + 无后处理 | 检测数最少(18px)，虚警率最低 |
| **光谱角度量** | **SAM** + Notch + med_open | 无需协方差逆矩阵，对场景变化鲁棒 |
| **多目标同时检测** | **MTICEM** + Notch + med_open | 内建多目标支持 |
